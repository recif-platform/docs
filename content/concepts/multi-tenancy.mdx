## Overview

Recif achieves multi-tenancy through Kubernetes namespaces. Each team gets its own namespace with dedicated resource quotas, network policies, RBAC roles, and governance rules. Context cascades from platform to team to agent to user.

## What You'll Learn

- Namespace-per-team isolation model
- RBAC: platform admin, team admin, member, viewer roles
- Resource quotas (CPU, memory, token budgets)
- Network policies for agent-to-tool communication
- Multi-level context hierarchy (platform, team, agent, user, session)
- Cross-agent user preference propagation
